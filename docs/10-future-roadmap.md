# Future Roadmap — Not Part of Current Implementation

These ideas are retained for future product planning only. They are intentionally excluded from the current extension runtime, React navigation, settings, message contracts, persistence, dependencies, task estimates, acceptance criteria, pilot scope, and release-readiness gates.

## Business Unit Intelligence Hub

A future optional capability for publishing approved reusable intelligence across products and teams, including cross-product pattern and integration discovery. It would require a separately approved security, identity, governance, deployment, and data-boundary design.

## Local Model and LoRA Adaptation

A future optional capability for approved local inference, organization-specific adapters, evaluation, and hybrid routing. It would require a separately approved runtime, model-license, dataset-governance, hardware, process-isolation, and trust design.

Neither capability is implemented or required by Keystone today. Current execution routes are deterministic Keystone services, capability-proven GitHub Copilot delegation, explicit manual user action, or an honest unsupported result.
